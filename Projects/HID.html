Sentinel-HID v1

Project by : Ehab thaer

بواسطة : ايهاب ثائر

ما هو Sentinel - HID؟

Sentinel-HID هو نظام بسيط وفعّال يهدف إلى رصد الأجهزة التي تُعرّف نفسها كأجهزة إدخال (مثل الكيبورد أو الماوس) عند توصيلها بالحاسوب عبر USB. الهدف الأساسي هو التقليل من خطر هجمات BadUSB، التي تعتمد على أجهزة ظاهرها طبيعي لكن تُبرمج على تنفيذ أوامر ضارة فور توصيلها.

ما هي مشكلة BadUSB؟

  • بعض أجهزة HID مثل لوحات المفاتيح المبرمجة، يمكن برمجتها لتنفيذ أوامر خبيثة بسرعة عند توصيلها.
  • هذه الأجهزة غالبًا لا تحتوي على ملفات، لذا لا يتم رصدها من مضادات الفيروسات.
  • يتم التعرف عليها من النظام كـ HID عادي، مما يصعب التمييز بينها وبين الكيبورد الحقيقي.

ماذا يفعل Sentinel-HID؟

يراقب أجهزة USB الموصولة ويركز على نوع HID بالتحديد، حيث يقوم بالآتي:

  • التحقق من كون الجهاز موثوق (من خلال whitelist).
  • مراقبة سرعة ضغط المفاتيح (Keystroke Rate).
  • إذا كانت السرعة مريبة، يتم فصل الجهاز فورًا باستخدام udev.
  • تسجيل ضغطات المفاتيح القادمة من الجهاز.
  • تسجيل كل أحداث الجهاز في Log مستقل.
  • إرسال تنبيه فوري عبر Telegram عند رصد نشاط مريب.

ما الذي يتم تسجيله؟

  • Keylogger File: يسجل ضغطات المفاتيح عند النشاط غير الطبيعي.
  • Activity Log File: يسجل الأحداث التالية:
    • وقت التوصيل
    • نوع الجهاز
    • Vendor ID و Product ID
    • تم السماح أو الحظر
    • سرعة الكتابة

المميزات باختصار

  • يعمل بدون واجهة رسومية
  • قواعد whitelist مرنة
  • تسجيل شامل للأحداث
  • تنبيهات Telegram فورية
  • لا يعتمد على أدوات خارجية
  • قابل للدمج مع أنظمة حماية أخرى

متى يكون مفيدًا؟

  • في الحواسيب الإدارية أو بيئات المختبر
  • في أنظمة Linux التي تفتقر للحماية الطرفية
  • لكشف وتوثيق التوصيلات المشبوهة

English Version

What is Sentinel-HID?

Sentinel-HID is a lightweight and effective system designed to detect devices that identify themselves as Human Interface Devices (like keyboards or mice) when connected via USB. The goal is to minimize the risk of BadUSB attacks, where seemingly normal devices execute malicious commands.

What’s the problem with BadUSB?

  • Some HID devices, like programmable keyboards, can be configured to run malicious scripts instantly after being plugged in.
  • They don’t usually contain files, so antivirus software can’t detect them.
  • The system sees them as regular keyboards, making them hard to distinguish from real ones.

What does Sentinel-HID do?

It monitors USB devices connected to the system and focuses on HID types. Main actions include:

  • Checking whether the device is trusted (using a whitelist).
  • Monitoring keystroke speed.
  • If the speed is suspicious (e.g., over 12 keys/sec), the device is disconnected using system-level tools (udev).
  • Keylogging activity from the suspicious device.
  • Logging all events related to the device in a separate log file.
  • Sending a Telegram alert if suspicious activity is detected.

What gets logged?

  • Keylogger File: Records only when unusual typing speed is detected.
  • Activity Log File: Includes:
    • Connection timestamp
    • Device type
    • Vendor ID & Product ID
    • Whether it was allowed or blocked
    • Keystroke speed if monitoring was active

Key Features

  • Works without GUI (ideal for servers/headless systems)
  • Flexible whitelist rules
  • Detailed event logging
  • Telegram instant alerts
  • No complex external tools required – just Python on Linux
  • Easily integrable with other security systems

When is it useful?

  • In environments with strict control over connected devices
  • On Linux systems without advanced endpoint protection
  • As a lightweight way to detect and document suspicious USB activity
